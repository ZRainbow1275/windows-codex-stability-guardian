# Security Policy

## Supported Releases

The latest tagged release is supported for security and stability fixes.

## Reporting a Vulnerability

Do not include local machine secrets, private prompts, session logs, or personal profile data in public issues.

If you discover a security issue:

1. Prepare a minimal reproduction using sanitized paths and redacted local data.
2. Use GitHub security reporting when available.
3. If private reporting is not available, open a GitHub issue with only the minimum non-sensitive evidence needed to reproduce the problem.

## Repository Privacy Boundary

This repository is intended to contain:

- source code
- public release notes
- sanitized documentation

This repository should not contain:

- personal email addresses in docs
- local workstation absolute paths tied to a real user profile
- private session transcripts
- machine-specific secrets or tokens
